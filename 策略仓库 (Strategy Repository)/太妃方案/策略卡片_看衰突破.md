# 🎯 策略概览

**一句话描述**:
在交易区间内,不看好突破 K 线能成功突破,反向入场做失败突破交易。

**Al Brooks 术语**: Fade, Failed Breakout, Trading Range Reversal  
**历史原始术语（原太妃）**: 看衰突破

---

## 📋 入场条件 (Entry Criteria)

### 市场背景要求

- [x] 市场周期: **交易区间** (明确的上下边界)
- [x] 区间特征: K 线数 ≥20 根,或回调深度>75%
- [x] 波段结构: 阴阳交错重叠,买卖压力相对充分

### 关键识别要素

1. **区间定义**:
   - 双边交易性格占主导
   - 价格在明确的支撑/阻力之间反复震荡
   - 突破倾向失败(回到区间内)
2. **区间类型识别**:
   - **小型区间**: <20 根 K 线,可能是趋势中的旗形回调
   - **大型区间**: ≥20 根 K 线,更可能是真正的均衡状态
   - **宽幅通道**: 虽有斜率但双边交易强烈
3. **突破失败倾向**:
   - 区间规模越大,突破失败概率越高
   - 价格接近区间边界时,反向订单积聚
   - 市场 80%时间处于区间,趋势只占 20%

### 信号棒特征

- [x] 位置: 区间上缘(做空)或下缘(做多)
- [x] K 线形态: 测试区间边界的 K 线
- [x] 确认: 反向 K 线出现,显示突破失败
- [x] 理想: 区间高度充足(≥2 倍止损空间)

### 特殊注意

- **空间要求**: 获利方向必须有充足空间(盈亏比 ≥1.5:1)
- **区间 vs 趋势**: 趋势中的旗形回调最终倾向顺势突破
- **均衡中轴**: 价格围绕 50%胜率中轴震荡,边界提供折返

## 🛡️ 风险管理 (Risk Management)

### 止损位置

- **标准**: 区间边界外 2-4 跳
  - 区间上缘做空: 止损在上缘上方
  - 区间下缘做多: 止损在下缘下方
- **保守**: 前一个波段极值外

### 目标位计算

- **首选**: 区间对侧(完整区间高度)
  - 上缘入场: 目标下缘
  - 下缘入场: 目标上缘
- **次选**: 区间 50%位置(中轴)
  - 适用于初次尝试或保守交易者
- **等距运动**: 突破成功后,目标为区间高度的等距投射

### 入场时机选择

- **激进**: 价格触及区间边界立即反向入场
  - 优点: 最佳盈亏比
  - 风险: 可能有第二次或第三次测试
- **保守**: 等待反向 K 线确认突破失败
  - 优点: 更高确认度
  - 缺点: 盈亏比略差

---

## ⚠️ 失败模式 (Failure Signals)

当出现以下情况,立即退出或不入场:

- [x] **突破成功**: 价格强势突破区间,远离边界
- [x] **区间缩小**: 形成更小的区间(收敛三角形)
- [x] **趋势脉络**: 在强趋势中途的旗形回调
- [x] **动能增强**: 突破 K 线动能强劲,连续突破
- [x] **空间不足**: 区间高度太小,盈亏比<1.5:1

### 区间 vs 旗形的判断

| 特征     | 交易区间 | 旗形回调   |
| -------- | -------- | ---------- |
| K 线数量 | ≥20 根   | <20 根     |
| 斜率     | 接近水平 | 明显斜率   |
| 宏观脉络 | 趋势结束 | 趋势中途   |
| 突破倾向 | 失败     | 成功(顺势) |

## 📊 Al Brooks 理论对应

### 核心概念映射

| 太妃术语 | Al Brooks 术语     | 说明             |
| -------- | ------------------ | ---------------- |
| 看衰突破 | Fade, Look to Fade | 不看好突破成功   |
| 区间     | Trading Range      | 双边交易地带     |
| 突破失败 | Failed Breakout    | 突破后回到区间内 |
| 折返机制 | Mean Reversion     | 价格向均衡回归   |

### 适用的 Al Brooks 形态

- **Trading Range**: 核心环境
- **Failed Breakout**: 核心形态
- **Limit Order Market**: 在边界使用限价单
- **Second Entry**: 第二次测试边界的入场

### 课程来源

- **时间码**: 12:13 - 看衰突破成功

### 关键要点

1. **术语来源**: Al Brooks 原文"look to fade",简中译本为"淡出趋势 K 线"
2. **行为本质**: 相当于与突破做对,投下"反对票"
3. **市场统计**: 80%时间在区间,20%时间在趋势
4. **循环关系**: 失衡 → 趋势 → 均衡 → 区间 → 新失衡
5. **空间至关重要**: 区间高度不够就不要交易

### 个人理解

- 看衰突破是区间交易的核心策略
- 与急赴磁体的区别: 看衰突破是在边界,急赴磁体是接近边界(≤25%)
- 关键是判断真区间还是旗形回调(看 K 线数和宏观脉络)
- Al Brooks 的 Trading Range 章节就是讲这个

## 🔄 操作检查清单 (Checklist)

**入场前检查**:

- [ ] 确认区间: K 线数 ≥20 根或回调深度>75%
- [ ] 识别边界: 上缘和下缘清晰可见
- [ ] 评估空间: 区间高度足够(盈亏比 ≥1.5:1)
- [ ] 判断类型: 不是趋势中的旗形回调
- [ ] 观察测试: 价格接近或触及边界

**持仓中监控**:

- [ ] 警惕突破成功: 价格强势远离边界则止损
- [ ] 观察向目标运动: 向区间对侧或 50%位运动
- [ ] 关注第二次测试: 可能提供加仓机会

**复盘问题**:

- [ ] 是否真的是区间(不是旗形)?
- [ ] 区间高度评估是否正确?
- [ ] 入场时机选择合理吗?
- [ ] 如果失败,是否有趋势脉络?

> [!tip] 实战建议
>
> 1. **新手**: 只交易大型区间(≥20 根 K 线),等待反向 K 线确认
> 2. **中级**: 可在边界直接反向入场,但要控制仓位
> 3. **高手**: 区分区间(高概率)vs 旗形(低概率),使用限价单

> [!warning] 风险提示
>
> - 区间高度不足(<2 倍止损)不要交易
> - 趋势中途的旗形回调最终倾向顺势突破
> - 连续多次测试边界后突破概率增加
> - 突破成功后可能有等距运动,及时止损
> - 避免在窄幅区间(<10 跳)内交易

```dataviewjs
const currentPatterns = dv.current().patterns_observed || [];
const currentStrategy = dv.current().strategy_name;

// 辅助函数：检查数组交集
function hasIntersection(arr1, arr2) {
    if (!arr1 || !arr2) return false;
    const a1 = Array.isArray(arr1) ? arr1 : [arr1];
    const a2 = Array.isArray(arr2) ? arr2 : [arr2];
    return a1.some(item1 => a2.some(item2 => item1.includes(item2) || item2.includes(item1)));
}

dv.table(["日期", "品种", "方向", "结果", "盈亏", "执行"],
    dv.pages('"Daily/Trades"')
    .where(p => {
        const tradePatterns = p.patterns_observed;
        const tradeStrategy = p.strategy_name;

        // 1. 策略名称完全匹配
        if (currentStrategy && tradeStrategy && tradeStrategy === currentStrategy) return true;

        // 2. 形态标签共振 (只要有一个形态重叠)
        if (hasIntersection(currentPatterns, tradePatterns)) return true;

        return false;
    })
    .sort(p => p.date, "desc")
    .map(p => [
        p.file.link,
        p.ticker,
        p.direction,
        p.outcome == "止盈 (Win)" ? "✅ 止盈" : (p.outcome == "止损 (Loss)" ? "❌ 止损" : "⚪ 平手"),
        p.net_profit,
        p.execution_quality
    ])
)
```
