# 策略卡片审核检查清单

> **目的**: 确保所有策略卡片遵循数据标准,维护系统一致性

## 使用方法

1. 创建或修改策略卡片后,逐项检查
2. 所有必需字段必须通过
3. 建议字段尽量完善
4. 发现问题立即修正

---

## ✅ 必需字段检查

### 1. 策略名称/strategy_name
- [ ] **有值**: 不能为空
- [ ] **唯一性**: 与其他策略不重复
- [ ] **格式**: 中文名 (English Name)
- [ ] **示例**: ✅ `20均线缺口 (20 EMA Gap)` ❌ `20均线缺口`

### 2. 方向/direction
- [ ] **必填**: 必须有值
- [ ] **标准值**: 必须是以下之一:
  - `做多 (Long)`
  - `做空 (Short)`
  - `双向 (Both)`
- [ ] **来源**: 从`Templates/属性值预设.md`复制
- [ ] **默认**: Al Brooks策略通常是`双向 (Both)`

### 3. 市场周期/market_cycle
- [ ] **至少一个**: 不能为空数组
- [ ] **标准值**: 必须来自预设列表:
  - 强趋势 (Strong Trend)
  - 弱趋势 (Weak Trend)
  - 交易区间 (Trading Range)
  - 紧凑交易区间 (Tight Trading Range)
  - 突破模式 (Breakout Mode)
  - 尖峰与通道 (Spike and Channel)
- [ ] **示例**: ✅ `- 强趋势 (Strong Trend)` ❌ `- 强趋势`

### 4. 设置类别/setup_category
- [ ] **有值**: 不能为空
- [ ] **标准值**: 必须是以下之一:
  - 趋势突破 (Trend Breakout)
  - 趋势回调 (Trend Pullback)
  - 趋势反转 (Reversal)
  - 区间逆势 (TR Fade)
- [ ] **来源**: 从`Templates/属性值预设.md`复制

### 5. 观察到的形态/patterns_observed
- [ ] **至少一个**: 不能为空数组
- [ ] **标准值**: 必须来自预设列表
- [ ] **对应关系**: 形态名称应与策略名称对应
- [ ] **示例**: 策略名"20均线缺口" → 形态包含"20均线缺口 (20 EMA Gap)"

---

## 🔧 可选字段检查

### 6. 时间周期/timeframe
- [ ] **标准值**(如有): 必须来自预设列表:
  - 1m / 5m / 15m / 1H / 4H / Daily
- [ ] **建议**: 标注策略最佳时间周期

### 7. 风险等级/risk_level
- [ ] **标准值**(如有): 必须是以下之一:
  - 低 (Low)
  - 中 (Medium)
  - 高 (High)
- [ ] **建议**: 根据盈亏比和胜率判断

### 8. 入场条件/entry_criteria
- [ ] **清晰明确**: 可执行的具体条件
- [ ] **可量化**: 避免模糊描述
- [ ] **示例**: ✅ "K线连续20-30根未触及EMA" ❌ "价格远离均线"

### 9. 止损建议/stop_loss_recommendation
- [ ] **具体位置**: 明确止损点
- [ ] **多个选项**: 保守/激进
- [ ] **示例**: ✅ "前一波段极值外2 ticks" ❌ "合理位置"

### 10. 目标建议/take_profit_recommendation
- [ ] **具体位置**: 明确目标点
- [ ] **多个选项**: 首要/延伸
- [ ] **示例**: ✅ "原趋势极值(Old High/Low)" ❌ "合适位置"

---

## 📊 内容质量检查

### 11. 策略简述
- [ ] **一句话描述**: 清晰概括策略核心
- [ ] **适用场景**: 说明何时使用
- [ ] **Al Brooks术语**: 标注对应的Brooks术语

### 12. 风险提示/risk_alerts
- [ ] **完整性**: 列出主要风险
- [ ] **可操作**: 说明如何识别风险
- [ ] **示例**: ✅ "K线数>40根时可能是大级别趋势" ❌ "有风险"

### 13. 盈亏比/risk_reward
- [ ] **有值**: 不能为空
- [ ] **合理范围**: 通常1:1 - 3:1
- [ ] **示例**: ✅ "2:1 - 3:1" ❌ "高"

---

## 🔍 一致性检查

### 14. 方向与描述一致
- [ ] **策略描述**: 与direction字段一致
- [ ] **入场条件**: 不应包含相反方向的描述
- [ ] **示例**: direction=做多,但描述中不应只提"卖出"

### 15. 市场周期与设置类别匹配
- [ ] **逻辑一致**: 
  - 强趋势 → 趋势突破/回调
  - 交易区间 → 区间逆势/反转
- [ ] **示例**: ❌ 强趋势 + 区间逆势(不匹配)

### 16. 形态与策略名称对应
- [ ] **主形态**: patterns_observed应包含策略名对应的形态
- [ ] **示例**: 策略"急赴磁体" → 形态包含"急赴磁体 (Rush to Magnet)"

### 17. 来源/source
- [ ] **有值**: 标注策略来源
- [ ] **格式**: 课程名称 + 链接(如有)
- [ ] **示例**: `Brooks Playbook（原太妃价格行为） L17B`

## 🚫 常见错误

### ❌ 错误1: 自创属性值
```yaml
方向/direction: 中性  # ❌ 不在预设列表中
```
**修正**:
```yaml
方向/direction:
  - 双向 (Both)  # ✅ 使用预设值
```

### ❌ 错误2: 缺少英文名
```yaml
策略名称/strategy_name: 20均线缺口  # ❌ 缺少英文
```
**修正**:
```yaml
策略名称/strategy_name: 20均线缺口 (20 EMA Gap)  # ✅
```

### ❌ 错误3: 空数组
```yaml
观察到的形态/patterns_observed: []  # ❌ 空数组
```
**修正**:
```yaml
观察到的形态/patterns_observed:
  - 20均线缺口 (20 EMA Gap)  # ✅ 至少一个
```

### ❌ 错误4: 格式不一致
```yaml
市场周期/market_cycle:
  - 强趋势  # ❌ 缺少英文
```
**修正**:
```yaml
市场周期/market_cycle:
  - 强趋势 (Strong Trend)  # ✅ 中英文格式
```

1. **创建阶段**: 使用`Templates/策略卡片规范模板.md`
