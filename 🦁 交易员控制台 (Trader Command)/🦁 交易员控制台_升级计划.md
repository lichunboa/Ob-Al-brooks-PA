# 🛠️ 交易员控制台 (Trader Command) 升级路线图

> [!abstract] 升级目标
> 将静态的笔记仓库转化为动态的“战术辅助系统”，实现**市场背景自动感知**、**策略规则强制提醒**以及**复盘自动化**。

## 🚀 第一阶段：态势感知 (Context Aware)
**目标**：解决“逆势操作”和“情绪化交易”问题。
- [ ] **功能 1.1：市场背景面板 (Market Context)**
    - [ ] 修改文件：`scripts/pa-view-trend.js`
    - [ ] 逻辑：自动读取最新 `Daily/` 日记中的 `market_cycle` 字段。
    - [ ] 显示：在控制台首页左侧显示今日是“牛市趋势”还是“震荡区间”。
- [ ] **功能 1.2：心态监控 (Mindset Monitor)**
    - [ ] 修改文件：`scripts/pa-view-trend.js`
    - [ ] 逻辑：统计最近 7 笔交易的 `management_error` 标签（如 FOMO/Tilt）。
    - [ ] 显示：红绿灯预警（状态极佳/有点起伏/极度危险）。

## 🧠 第二阶段：策略链接 (Strategy Link)
**目标**：解决“规则遗忘”问题，盘中实时查阅“说明书”。
- [ ] **功能 2.1：策略超链接**
    - [ ] 修改文件：`scripts/pa-view-strategy.js`
    - [ ] 逻辑：将策略名称（如 MTR）转化为点击事件，直接打开 `Notes 笔记/` 下的对应文件。
- [ ] **功能 2.2：AI 资金教练**
    - [ ] 修改文件：`scripts/pa-view-strategy.js`
    - [ ] 逻辑：根据实盘资金曲线（正/负），动态给出建议（继续保持/暂停实盘回模拟）。

## ⚡ 第三阶段：自动化 (Automation)
**目标**：解决“复盘繁琐”问题。
- [ ] **功能 3.1：一键生成日记 (待开发)**
    - [ ] 目标：在控制台输入简码（如 `ES, Long, MTR, +5`），自动创建标准化的 Markdown 交易日记。
