# Optimization Patterns (优化模式库)

## 成功模式

### 模式 1: 纯函数提取

**适用场景**: 组件中有大量计算逻辑

**步骤**:
1. 识别纯函数(无副作用)
2. 提取到 `src/utils/`
3. 添加单元测试
4. 在原组件中引用

**优点**:
- 风险低
- 易于测试
- 提高代码复用性

---

### 模式 2: Hook 提取

**适用场景**: 组件中有复杂的状态管理

**步骤**:
1. 设计 Hook 接口
2. 提取到 `src/hooks/`
3. 保持数据流一致
4. 独立测试

**优点**:
- 分离关注点
- 提高可维护性
- 便于复用

## 最佳实践

### 1. 小步快跑
- 每次只做一个小改动
- 立即测试
- 立即提交

### 2. 保持可回滚
- 每次修改前创建备份分支
- 使用有意义的提交信息
- 频繁提交

### 3. 充分验证
- 构建测试
- 功能测试
- 性能测试
