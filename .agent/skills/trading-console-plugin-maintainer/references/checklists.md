# Checklists（可复制执行）

## UI 整理（不动功能）

- 明确用户要求的 IA 变化（入口移动/合并/删除 tab）
- 明确“不做什么”（不新增模块、不改逻辑、不加交互）
- 选定稳定锚点（标题/分隔符/模块头）
- 小步改动：一次只做一个可验证动作
- 全局搜索验证：
  - 关键词命中数符合预期（例如 `✅ 每日行动` 只剩 1 次）
  - 被移除页面标识清零（例如 `"daily"` 不再出现）
- 构建门禁：`cd .obsidian/plugins/al-brooks-console && npm run build`

## Dashboard.tsx 大文件安全编辑

- 先全局搜索 1–3 个锚点关键词（模块标题）
- 每个删除/替换操作都必须：
  - 有明确起止边界（前后各有稳定锚点）
  - 不跨越组件定义边界（不要越过 `return (` / `</...>`）
- 每完成一个删除动作：
  - 立刻跑 `npm run build`
  - 若失败，优先回到最近一次通过的状态再重来

## 集成（Tasks 等外部能力）

- 调用前必须 `can("...")`
- 缺失能力时必须降级显示提示（而不是静默隐藏或报错）
- 变更后检查：页面渲染不崩溃、按钮点击无异常日志
